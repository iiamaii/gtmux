01 · Note

Note

Pinned annotation block — used to tag a shape, sketch, or region with a figure number and a short editorial remark. Accent rail on the left ties it to whatever it annotates. Collapses to a single icon chip when minimized.

Anatomy
  • 1
    Accent rail — 2px left border in --accent. Replaces a full border so the card reads as attached, not floating. Persists in the minimized chip.
  • 2
    Note-head (18px) — fixed-height mono strip. Grid: glyph · label · meta · action cluster. 9.5px / 0.6px tracking, --muted.
  • 3
    Note glyph (12px) — speech-bubble with one line, in --accent. Signals "annotation" and ties to the layer-tree icon.
  • 4
    Label — <b>Fig. NN</b> · Topic. Bold is --fg / 540 / 0.4px; topic is --muted.
  • 5
    Meta — pushed right with margin-left:auto. Author initials, edit time, or both. Drops on narrow widths.
  • 6
    Action cluster — two 18×18 ghost icons in a .note-actions wrapper. Opacity 0 → 1 on hover. Order: minimize · close. Close hover flips to #e5484d.
  • 7
    Note-body — sans, 12px / 1.4. One paragraph. If you need two — switch to a Document.
  • 8
    Chip glyph (14px) — shown only when .is-min. 9×9 square outline with a faint horizontal line — reads as a folded note token. Whole chip is the click target to restore.

04 · Panel

Panel — Terminal

A windowed container for live content. Renders its own chrome (title · status · min/max/settings/close) and hosts a fullscreen-dark terminal as canonical content. Behaves like a movable window on the canvas. Until a terminal is attached, the body shows a centered "Not connected" empty state with a Connect CTA — never blank, never fake output.

Anatomy
  • 1
    Frame — --surface-2 base (visible only while loading), 1px --border, radius 6px, --shadow-sm. overflow: hidden clips overlay rounding.
  • 2
    Img-frame — absolute fill. background-size: cover · background-position: center. Real implementation swaps the CSS background for an <img> or canvas-rendered bitmap.
  • 3
    Alt pill (top-left, 18px) — frosted-glass capsule, mono 9px uppercase. Default state hidden until hover; the .warn variant (alt missing) is always visible in #e5484d.
  • 4
    Action cluster (top-right) — 22×22 frosted ghost buttons. Order: Replace · Fit · ⋯ · Close. Background rgba(0,0,0,.45) + backdrop-filter: blur(8px). Opacity 0 → 1 on container hover. Close hover flips to rgba(229,72,77,.92) matching the warn pill.
  • 5
    Caption overlay — bottom-anchored, linear-gradient(transparent → rgba(0,0,0,.55)) across 28→10px padding. Always white text — readable on any underlying image.
  • 6
    Filename — mono 10px / 540 / #fff. Truncates with ellipsis. Always shows extension.
  • 7
    Dimensions — mono 10px opacity: .7. Native pixel size, no zoom math.
  • 8
    Format · size (right) — pinned right with margin-left:auto. JPG · 412 KB pattern.
  • 9
    Empty state — when no image is loaded: .is-empty swaps the frame for a dashed drop-zone with a centered "Empty" mono label. On hover the label fades out and a bare + glyph (22px stroke, --accent, no fill) scales in from 0.85 → 1.0. The whole shape is clickable — the + is a visual handle, not a separate target.

07 · Shared rules

Shared rules

Apply to the five canvas-item components (note · document · file path · panel · image). The color picker is editor chrome, not a canvas item — only the theme rule (D) applies to it.

cross-cut
Behaviour
  • A
    Outer frame — all five canvas items sit inside .canvas-item (absolute on the workspace). Width / height pinned in style; the shape fills 100%. The color picker is the exception: it's an inspector popover with a fixed 240px width and content-driven height.
  • B
    Selection — when selected, the canvas adds a 1.5px --accent outline around .canvas-item, not on the shape. Shapes don't manage selection state.
  • C
    Hover — no per-shape hover effect. Hover state is on the canvas item wrapper (slight outline). Keeps content quiet.
  • D
    Theme — every token is theme-reactive. Don't hard-code colours in shape styles except language badges and the terminal body (which is intentionally dark on both themes).
  • E
    Layer tree — shows the type + name. Note: speech-bubble + line (canvas glyph) / square card (minimized chip glyph). Document: doc. Filepath: doc with </>. Panel: terminal. Image: mountain + sun.
  • G
    Close affordance — every canvas item exposes a close (×) button. Visible always on Panel/Document (window-like chrome) · hover-revealed on Note/Filepath/Image (content-first). Close hover always flips bg to #e5484d. Removal is handled by the canvas-level undo stack, not the shape.
  • H
    Empty / unloaded data — Document and Image both expose an .is-empty idle that shows a centered "Empty" mono label. On hover, the label fades out and a circular + button (36×36, --accent fill, white plus, soft drop-shadow) scales in from 0.85 → 1.0 — the + is the explicit affordance, but the whole shape stays clickable. Image additionally tints the frame border solid --accent with a 6% accent fill; Document keeps its file-strip head but mutes the filename.
  • F
    Overflow — all shapes use overflow: hidden. Long copy is the author's problem, not the shape's. Image fills the frame with cover.
