3.1 reconnectGate 8-state (ADR-0019 D5.4)

Page entry blocking gate — canMountApp 게이트로 본 화면 mount 차단/허용 결정.

booting (entry) attaching / hydrating ready (canMountApp) in_use / not_found / unreachable

4.3 multi-webpage Terminal mirror (ADR-0021 D1 + D10.3)

두 webpage 가 같은 terminal_id 를 mirror 하다가 죽고, 거의 동시에 respawn 시도. per-UUID Mutex 가 한 caller 만 실제 spawn, 다른 caller 는 idempotent reused:true.

D10.3 invariant: 동시 2+ 요청 시 한 caller 만 kill+spawn, 나머지는 reused:true short-circuit. UUID 당 alive PaneId 1 보장. PTY leak 0.

Browser-side 예외 8 시나리오

시나리오처리
새로고침 (F5 / Cmd+R)hint 있으면 reconnectGate.start(name), 없으면 booting → idle → AuthDialog
탭 background → foregroundvisibilitychange + heartbeat.isIdle 시 silentReattach. perception 0
OS sleep / wakeWS reconnecting → open 시 silentReattach. 200 idempotent 또는 409
Network bliplib/ws/client.ts 1s grace + exp backoff (1/2/4/8/16s)
Network drop (>30s)BE heartbeat timeout → flock release. 복귀 시 silentReattach 200 또는 409
Browser crash / OS killBE 30s timeout 정리. 다음 webpage 진입 시 정상
탭 닫기 (정상)beforeunload → sendBeacon('/api/leave') best-effort
여러 탭 동시 운영각 탭 = 다른 session. 같은 session 두 번째 탭 = 409 + "in use" badge
